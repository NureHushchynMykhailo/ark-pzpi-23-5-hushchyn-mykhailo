3 ОПИС ВИКОНАНОЇ РОБОТИ

     У ході виконання лабораторної роботи №1 було створено базовий документ Vision & Scope (Додаток Б) для програмної системи «Моніторинг забруднення водних ресурсів». Цей документ закладає основу для подальшої розробки та аналізу архітектури, чітко визначаючи бізнес-проблему, бачення рішення, його функціональні межі та технологічний стек.
     Метою роботи було визначити актуальну проблему, описати бачення інтелектуального рішення на базі IoT/веб-технологій, сформулювати бізнес-логіку, функціональність ключових компонентів та обрати програмно-технологічну базу.
     Було виконано такі кроки:
     • Аналіз проблеми: проведено аналіз проблеми зростаючого забруднення водних об'єктів (річок, озер), складності та високої вартості традиційного лабораторного контролю, а також потреби в отриманні оперативних даних про якість води в реальному часі.
     • Аналіз аналогів: проаналізовано існуючі системи (наприклад, стаціонарні станції моніторингу або ручні тестери), виявлено їхні недоліки: висока ціна, низька мобільність, відсутність автоматизованого дистанційного оповіщення та інтеграції даних.
     • Формування бачення системи: сформульовано бачення системи як мобільної, масштабованої, інтелектуальної платформи, що забезпечує безперервний, автоматичний моніторинг ключових показників якості води (pH, температура, рівень кисню, провідність) та оперативне сповіщення про перевищення критичних норм.
     • Визначення бізнес-логіки: визначено ключові бізнес-процеси, включаючи: збір даних із датчиків, передачу та агрегацію інформації на сервері, автоматичний розрахунок індексу якості води (ІКВ), візуалізацію даних (карти, графіки) та автоматичне реагування на перевищення критичних норм (наприклад, активацію актуаторів для аерації води, подачі реагентів або інших заходів для нормалізації показників).
     • Опис функціональності компонентів: IoT-клієнт: збір даних, локальне кешування, захищена передача на сервер. Серверна частина: прийом даних, зберігання, обробка, розрахунок аналітичних показників, керування користувачами (REST API). Веб/Мобільний клієнт: візуалізація даних на інтерактивній мапі, налаштування сповіщень, перегляд історії вимірювань та аналітичних звітів. Актуатори та вплив на навколишнє середовище: передбачено інтеграцію актуаторів, які можуть виконувати автоматичні дії на основі отриманих даних (наприклад, включення систем аерації води при низькому рівні кисню, регулювання фільтрації або подачі реагентів для нейтралізації забруднень). Це дозволяє не лише контролювати якість води, але й безпосередньо впливати на її стан у реальному часі, підвищуючи ефективність управління водними ресурсами.
     • Обрано стек технологій: Node.js + Express для серверної частини, React для веб-клієнта, Kotlin (Android Studio) для мобільного застосунку, ESP32 або Raspberry Pi для IoT-модуля, PostgreSQL як базу даних, GitHub для контролю версій, Postman для тестування API;
	Таким чином, отримано завершений документ Vision & Scope, що слугує основою для подальшої розробки, визначення конкретних вимог та детального архітектурного проектування системи моніторингу водних ресурсів.

1.1. Аналіз проблеми
    У сучасному світі зростаюче забруднення річок, озер та інших водних ресурсів створює значну загрозу для екології та здоров'я населення. Традиційні методи контролю якості води, що базуються на періодичному ручному заборі проб та лабораторному аналізі, є надто повільними, дорогими та нездатними забезпечити оперативне реагування на аварійні чи раптові забруднення. Через це органи екологічного нагляду, фермерські господарства та громадські організації не отримують даних у реальному часі, що критично ускладнює своєчасне виявлення джерел забруднення та вжиття заходів щодо їх усунення. Існує нагальна потреба у створенні доступної, автоматизованої та інтелектуальної системи, яка зможе безперервно моніторити ключові показники якості води (такі як pH, температура, провідність, розчинений кисень) і миттєво сповіщати користувачів про будь-які відхилення від норми.
    Крім моніторингу, сучасні системи можуть також активно впливати на стан води через інтегровані заходи. Це дозволяє не лише своєчасно інформувати користувачів, а й безпосередньо зменшувати негативний вплив забруднень, підвищуючи ефективність управління водними ресурсами та забезпечуючи безпеку екологічної системи.
 1.2. Аналіз аналогів
    На ринку моніторингу якості води наявні кілька груп рішень, проте кожна з них має суттєві обмеження, які наша система прагне подолати. Існують професійні комерційні станції, такі як обладнання від Xylem чи Hach. Це високоточні стаціонарні системи, які, однак, є надзвичайно дорогими, іноді коштуючи десятки тисяч доларів, що робить їх недоступними для малих підприємств, фермерських господарств чи громадських екологічних ініціатив. Крім того, вони часто використовують пропрієтарне програмне забезпечення та закриті протоколи, що ускладнює інтеграцію їхніх даних у власні ІТ-системи чи мобільні додатки. 	
    Іншим традиційним підходом є використання мобільних (ручних) тестерів та подальший лабораторний аналіз проб. Цей метод є застарілим, оскільки дані про забруднення з'являються лише через кілька днів, унеможливлюючи оперативне реагування на аварійні ситуації. Він забезпечує лише дискретну інформацію про одну точку в один момент часу і вимагає значних трудових витрат на постійний відбір проб. 
    Третя група — це бюджетні DIY-рішення, створені на базі Arduino або Raspberry Pi. Вони є економічно вигідними, але критично ненадійними: не мають належного захисту від навколишнього середовища, їхнє програмне забезпечення зазвичай обмежене простою передачею даних без повноцінної бізнес-логіки, такої як розрахунок Інтегрального показника якості води (ІКВ) чи професійної системи сповіщень. 
    Наша ж система «WaterQualityMonitor» усуває ці недоліки, поєднуючи економічну ефективність IoT-платформ (ESP32/Raspberry Pi) з надійністю, відкритою архітектурою (REST API на Node.js/Express) та професійними клієнтськими інтерфейсами (React, Kotlin), забезпечуючи безперервний, оперативний та доступний моніторинг.

1.3. Опис призначення системи
    Програмна система «Моніторинг забруднення водних ресурсів» — це інтелектуальна, масштабована екосистема, призначена для автоматичного, безперервного та віддаленого контролю фізико-хімічних показників якості води у визначених водних об’єктах (річки, озера, водосховища).
    
    Система використовує мережу автономних IoT-станцій, оснащених сенсорами, для збору даних про pH, температуру, провідність, розчинений кисень та інші ключові параметри в режимі реального часу. Ці дані агрегуються на центральному сервері, де автоматично аналізуються та порівнюються з екологічними нормами та пороговими значеннями.
    Основна цінність системи полягає у забезпеченні оперативності та прозорості даних. Це дозволяє користувачам — екологам, комунальним службам, промисловим підприємствам та громадським організаціям — миттєво отримувати сповіщення про критичні відхилення, проактивно реагувати на забруднення, локалізувати його джерела та мінімізувати екологічну шкоду.
    Бачення системи полягає у створенні надійної, доступної та легко розгортаної мережі моніторингу, яка трансформує традиційний, повільний контроль у сучасний, інтелектуальний інструмент для захисту водних ресурсів, сприяючи формуванню прозорої екологічної інформації та підвищенню екологічної безпеки.
    Крім моніторингу, система передбачає активне впливання на стан води за допомогою інтегрованих актуаторів. Наприклад, при перевищенні критичних норм розчиненого кисню або виявленні хімічного забруднення система може автоматично активувати аераційні установки, системи фільтрації або подачу нейтралізуючих реагентів. Це дозволяє не лише отримувати дані, а й безпосередньо регулювати якість води, зменшуючи шкоду для екосистеми та підвищуючи ефективність управління водними ресурсами.
    1.4 Опис бізнес-логіки
    Бізнес-логіка системи «Моніторинг забруднення водних ресурсів» є розподіленою та базується на чіткому поділі відповідальності між трьома ключовими компонентами: IoT-клієнтом, серверною частиною (бекендом) та адмініструванням.
    IoT-клієнт (ESP32 або Raspberry Pi) відповідає за фізичну взаємодію з середовищем:
     • Збір даних: автоматичний періодичний запуск сенсорів для зчитування показників якості води (pH, температура, провідність).
     • Локальна валідація: здійснення базової перевірки цілісності та діапазону даних (наприклад, відсікання значень, які фізично неможливі).
     • Управління зв'язком: захищена передача даних на сервер через HTTP. У разі втрати зв'язку, IoT-клієнт кешує дані (автономний режим) і синхронізує їх при відновленні мережі.
     • Оновлення конфігурації: прийом команд від серверу щодо зміни частоти опитування сенсорів або оновлення прошивки.
     Сервер є ядром системи і відповідає за обробку, аналіз, зберігання та інтеграцію:
     • Аутентифікація та авторизація: керування доступом користувачів (адміністратори, звичайні користувачі) та їхніми ролями.
     • Обробка вхідних даних: прийом, валідація та очищення сирих даних, що надходять від IoT-станцій, та їхнє структуроване збереження (SQLite/Firebase).
     • Аналіз даних та тригери: виконання складної бізнес-логіки, включаючи розрахунок Інтегрального показника якості води (ІКВ). Порівняння розрахованих значень із критичними порогами, визначеними адміністратором.
     • Генерація сповіщень: у разі перевищення порогів, ініціалізація механізму сповіщень (наприклад, відправка повідомлень через e-mail).
     • REST API для клієнтів: надання уніфікованого інтерфейсу для веб- (React) та мобільних (Kotlin) клієнтів для отримання історичних даних, графіків та статусів станцій.
     • Активне впливання на стан води: у разі виявлення критичних значень система може ініціювати роботу актуаторів - запуск аераційних установок, систем фільтрації або подачу реагентів для нейтралізації забруднень. 
Адміністрування охоплює всі функції, необхідні для підтримки та конфігурації екосистеми:
     • Управління користувачами: додавання/видалення користувачів, надання/скасування прав доступу та ролей.
     • Управління станціями: реєстрація нових IoT-станцій, прив'язка їх до географічних координат, моніторинг їхнього технічного стану (рівень заряду, час останнього з'єднання).
     • Конфігурація порогів: встановлення та зміна критичних і попереджувальних значень для кожного параметра (pH, T, O2), індивідуально для кожної станції або для регіону.
     • Аудит та звітність: генерація детальних звітів про якість води та журнали подій для контролюючих органів.
     Система розробляється з урахуванням можливості підтримки багатьох мов. Усі текстові елементи та інтерфейси (веб та мобільний) будуть відокремлені у зовнішні файли ресурсів. Це забезпечить гнучке додавання нових мов у майбутньому без зміни логіки коду. 
     На початковому етапі розробки буде підтримуватися як мінімум українська та англійська мови. Також буде забезпечено локалізацію числових форматів (десяткові роздільники), одиниць вимірювання (наприклад, Цельсій) та форматів дати/часу відповідно до регіональних стандартів користувача.

3.1. Опис функціональності серверної частини та IoT-клієнта
     Функціональність системи «Моніторинг забруднення водних ресурсів» розподілена між автономним IoT-клієнтом та центральним сервером. IoT-клієнт, що використовує платформу ESP32 або Raspberry Pi, відповідає за фізичну взаємодію з навколишнім середовищем. Його основна функція полягає у зборі даних, що включає регулярне зчитування показників з усіх підключених датчиків, їх локальну попередню обробку, включаючи калібрування та фільтрацію шумів, а також перетворення у стандартизовані одиниці вимірювання. У сфері комунікації IoT-клієнт забезпечує передачу даних на сервер через захищені HTTP-запити, управління зв’язком та автоматичне відновлення підключення у разі його втрати. Критично важливою є здатність працювати в автономному режимі, зберігаючи дані у локальній пам’яті та синхронізуючи їх із сервером після відновлення мережі. Крім того, IoT-клієнт приймає команди від серверу для зміни частоти опитування сенсорів або оновлення прошивки.
     Серверна частина, реалізована на Node.js та Express, виступає ядром системи та забезпечує прийом, валідацію та автентифікацію всіх запитів через REST API. Сервер відповідає за управління базою даних (PostgreSQL), де зберігаються історичні дані вимірювань, метадані станцій та облікові записи користувачів, а також за резервування даних. У сфері аналітики та бізнес-логіки сервер виконує складні алгоритми, включаючи розрахунок Інтегрального показника якості води (ІКВ), безперервно моніторить дані та порівнює їх із критичними порогами. У разі перевищення порогів сервер ініціює генерацію сповіщень, відправляючи повідомлення або e-mail користувачам. Система підтримує активне впливання на стан води за допомогою актуаторів IoT-клієнтів: при виявленні критичних значень може автоматично запускати аераційні установки, системи фільтрації або подавати реагенти для нейтралізації забруднень, що дозволяє не лише інформувати користувачів, а й безпосередньо стабілізувати якість води. Сервер також забезпечує управління доступом, автентифікуючи користувачів і перевіряючи їхні права перед наданням доступу до ресурсів системи.
3.2. Опис функціональності веб-клієнта та мобільного клієнта
     Веб-клієнт, розроблений на React, слугує основним інструментом для адміністрування та глибокого аналізу даних системи. Користувачі можуть переглядати детальні графіки історичних даних, включно з трендами всіх вимірюваних параметрів, таких як pH та Інтегральний показник якості води (ІКВ), з можливістю гнучкої фільтрації за часом. У частині адміністрування веб-клієнт дозволяє реєструвати нові IoT-модулі, керувати метаданими станцій та встановлювати і змінювати порогові значення для автоматичного спрацьовування тригерів, що відповідають за сповіщення та активізацію актуаторів. Крім того, веб-інтерфейс забезпечує потужну звітність, дозволяючи експортувати аналітичні дані про якість води у форматах CSV або PDF для подальшого використання контролюючими органами.
     Мобільний клієнт, розроблений на Kotlin в Android Studio, призначений для швидкого, віддаленого моніторингу та оперативного реагування. Його головний дашборд надає миттєвий огляд ключових показників якості води та статусу найближчої станції. Найважливішою функцією є сповіщення, які миттєво інформують користувача про перевищення критичних порогів, що може ініціювати як сповіщення, так і активізацію актуаторів для стабілізації стану води. Мобільний застосунок також дозволяє переглядати скорочену історію вимірювань та журнал останніх тривог, а також здійснювати віддалену діагностику технічного стану IoT-модулів, наприклад, перевіряти рівень заряду батареї.
     Обидва клієнти взаємодіють із сервером через REST API, забезпечуючи синхронний та актуальний обмін даними, що гарантує своєчасну аналітику та оперативне реагування як на інформаційні сповіщення, так і на активне регулювання стану водних ресурсів за допомогою актуаторів.
     
4.3 Опис програмних засобів та інструментів

    Розробка програмної системи для моніторингу забруднення водних ресурсів базуватиметься на сучасному, відкритому та ефективному технологічному стеку. Серверна частина (Backend), яка відповідає за обробку даних, аналітику та забезпечення API-доступу, реалізується на платформі Node.js у поєднанні з фреймворком Express.js, що дозволяє швидко створювати REST API для обміну даними між усіма компонентами системи. Для IoT-модуля, який функціонує як автономна станція збору даних, застосовуються мікроконтролери ESP32 або одноплатні комп’ютери Raspberry Pi, що забезпечують гнучкість роботи з різними сенсорами та можливість інтеграції актуаторів для активного впливу на стан води.
    Основною базою даних для зберігання історичних вимірювань та конфігурацій обрано PostgreSQL, що гарантує надійне зберігання структурованих даних, підтримку складних запитів та масштабованість системи. Для початкових етапів або для невеликих експериментальних розгортань можна використовувати SQLite, а для хмарної синхронізації даних — Firebase.
    Веб-клієнт, призначений для детальної аналітики та адміністрування, реалізується за допомогою бібліотеки React на JavaScript, що забезпечує швидку роботу інтерфейсу та його адаптивність. Мобільний клієнт розробляється мовою Kotlin у середовищі Android Studio, що дозволяє створити нативний, швидкий і функціональний застосунок з підтримкою Push-сповіщень та віддаленого моніторингу.
    Для контролю версій та організації командної розробки використовується система Git з хостингом на GitHub, а для тестування та валідації REST API застосовується інструмент Postman. Обраний стек технологій забезпечує надійність, гнучкість, простоту впровадження та можливість подальшого розширення системи, включно з інтеграцією нових датчиків, актуаторів та додаткових клієнтських інтерфейсів.
